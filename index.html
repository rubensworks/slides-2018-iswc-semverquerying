from core knowledge

What happens when this core knowledge is versioned?

Versioned: different versions of a dataset can exist next to each other.

Stream reasoning is related to this,
but focuses on versions with a linear temporal relation.

RDF Versioning

RDF Version (i)

Set of triples annotated with version label i

Query atoms

  • Version Materialization (VM):

    Which cities existed in (DBpedia) 2016?


    Query a single version
  • Delta Materialization (DM):

    Which cities were created or
    removed between 2015 and 2016?


    Query differences between 2 versions
  • Version Query (VQ):

    In which years has Brussels existed?


    Query all versions,
    results are annotated with their versions

VM, DM and VQ are structural query atoms.

They do not take into account the semantics of the data.

Example: Semantic Delta Materialization

Version 0: ex:Bob a ex:Animal. ex:Bob foaf:name "Bob". Version 1: ex:Bob a ex:Cat. ex:Bob foaf:name "Bob". Language: ex:Cat rdf:subClassOf ex:Animal.

Delta Materialization

- ex:Bob a ex:Animal. + ex:Bob a ex:Cat.

Semantic Delta Materialization

+ ex:Bob a ex:Cat.

Semantic query atoms

Semantic extension of the regular query atoms

  • Semantic Version Materialization (S-VM):

    Query over the semantic closure of a single version
  • Semantic Delta Materialization (S-DM):

    Query differences between the semantic closure of 2 versions
  • Semantic Version Query (S-VQ):

    Query the semantic closure of all versions,
    results are annotated with their versions
